Complete Figma UI/UX Design Prompt: Makati Milk Bank Management System (MMBMS)🌐 Project Context & Workspace SetupSystem Name: Makati Milk Bank Management System (MMBMS)   Platform Type: Web-Based Desktop Application. The design layout must use a standard 1440px desktop grid, optimized for standard desktop browsers (Chrome, Safari, Firefox) used in clinical offices and hospital workstations.  Overall Goal: Replace manual logbooks with a sleek, high-biovigilance web dashboard that safely monitors the end-to-end lifecycle of donated breast milk.  🎨 Color Palette & UI Design Tokens (Blue & Pink Theme)The application requires a professional, clean medical aesthetic that feels highly clinical yet human and accessible. Use the following design tokens:1. Brand & Structural ColorsPrimary/Clinical Blue (#1E3A8A or equivalent): Represents authority, security, and healthcare data integrity. Use this for the fixed left navigation sidebar, main table headers, card borders, and primary administrative action buttons (e.g., "Save to Database").Accent/Soft Pink (#EC4899 or equivalent): Represents care, maternal donation, and community. Use this for donor engagement buttons (e.g., "Register New Mother"), active sidebar navigation states, and interactive toggles.Neutral App Backgrounds: Clean canvas white (#FFFFFF) and ultra-light gray-blue (#F8FAFC) for page backgrounds, keeping data tables exceptionally crisp and legible.2. Functional Medical Status ColorsTo ensure the theme does not conflict with medical safety tracking, use carefully mapped semantic variants for status badges:Raw Milk Inventory: Light Pastel Pink badge with dark pink text (representing fresh, unpasteurized community donations).  Quarantined Batch: Soft Amber / Muted Yellow badge (standard clinical warning color for pending lab work).  Pasteurized / Passed Inventory: Soft Teal / Cyan Blue badge with dark blue text (indicating the milk has passed pasteurization and safety protocols and is ready for dispensing).  Rejected / Alert Flag: Muted Coral Red (used strictly for failed lab results or when exceeding daily extraction safe thresholds).  🧭 Global Navigation Layout (Role-Based Access Control)Design a Fixed Left Sidebar Navigation Layout with an integrated upper-right Profile Menu. The items in the sidebar should dynamically hide or show based on the logged-in user profile:  Administrator / Manager Workspace: Access to Dashboard Overview, User Account Management, System Audit Logs, and DOH Report Generation.  Nurse / Staff Workspace: Access to Donor Management, Milk Collection Log, Pasteurization & Lab Board, and Beneficiary Dispensing Portal.  Donor Portal View: Highly simplified client-facing dashboard to view personal profile details, update ongoing health questionnaires, and track donation status.  Beneficiary Portal View: Simplified portal for mothers/guardians to check milk type availability and view active pickup schedules.  🖥️ Screen-by-Screen UI/UX SpecificationsScreen 1: The Login Portal (UC-01)Layout: Centered vertical card layout on a soft, light blue-gray background.  Input Fields: Username (text) and Password (hidden/dot text) with light blue borders.  Focus State: Active field borders glow in Soft Pink.Primary CTA: A prominent Clinical Blue button labeled "Sign In to MMBMS".  Screen 2: Donor Management Directory (FR-02)Top Header Bar: Contains a Soft Pink button with a + icon labeled "Register New Donor Profile".  Registration Slide-Out Modal (Drawer): Text inputs for Full Name, Behavioral Screening questionnaire, Health Status Toggle, and a mandatory Consent Form checkbox.  Main Data Table: Columns for Donor Tracking Number (DTN), Name, and Registration Date.  Critical Component State: Any donor marked with a temporary medical deferral must feature a high-visibility, light-coral row highlight or badge.  Screen 3: Milk Collection Station (FR-03)Layout: Form-based desktop view optimized for fast data entry.  Input Field Elements:DTN Selector (Dropdown with instant search functionality).  Collection Tracking Number (CTN) field.  Volume Input field (numeric with standard "mL" suffix suffix label).  Arrival Temperature Input field (numeric with "°C" suffix label).  Collection Program Radio Button Selector: Three explicit options: Supsup Todo (Mobile Center Collection), Milky Way (Hospital NICU), or Mom's Act (Household Pickup).  UX Safety Prompt: Create a popup warning overlay in Figma showing the system's guardrail trigger if an operator accidentally types an input exceeding 800mL per day or outside the safe session range of 30–240mL.  Screen 4: Processing, Pasteurization & Lab Board (FR-04)Layout: Split-screen or Kanban Board layout tracking batches from Raw to Approved.Pasteurization Logger: Input fields for the Holder Method logging variables: Time-In, Time-Out, and specific thermal controls tracking the 62.5°C to 63°C pasteurization range.  Microbial Lab Testing Panel: Explicit outcome toggle buttons: Pass, Quarantine, or Reject.  Strict Prototyping Constraint: In your Figma interaction flow, the batch status pill cannot switch to the Teal/Cyan "Ready for Dispensing" style unless the Pass button is explicitly selected.  Screen 5: Real-Time Inventory Dashboard (FR-05)Top Summary KPI Cards: Three large metric cards styled with subtle gradient backgrounds blending from Clinical Blue to Soft Pink at 10% opacity. They show total volumes for: Raw Stock, Quarantined Stock, and Ready Pasteurized Stock.  Inventory Data Grid: Detailed columns displaying Batch ID, Expiration Date (auto-calculated 6–12 months window), and Status Badge.  System Alert Element: Design a bold banner that flashes near the top layout warning staff when specific milk volume types are under a "Low-Stock Alert" threshold.  Screen 6: Beneficiary Portal & Dispensing Counter (FR-06, FR-07)Beneficiary Profile Section: Fields logging Guardian Information, Infant Clinical Needs, and an interactive file attachment link showing verified Pediatrician Prescriptions/Clinical Abstracts.  Priority Tag Component: Distinct multi-colored tag pills ranking infant medical urgency (e.g., Priority 1: Critical NICU Preemie).  Dispensing Form Interface: Includes processing fee automatic calculations, a checkbox confirming safety clearance verification, and a "Release Milk" primary button that triggers a simulated stock volume deduction.  Screen 7: Automated SMS Notification Logs (FR-06)Layout: Clean communication activity thread.Elements: A view displaying outgoing automated SMS logs sent to waiting guardians when required milk types change to available.  Interactive Element: A detailed delivery status column indicating Sent, Delivered, or Failed (Retry Required) to establish full message tracking reliability.  Screen 8: Administrative Analytics & DOH Reports (FR-08)Control Bar: Dropdowns allowing the user to switch reporting views between Daily, Weekly, Monthly, and Annual parameters.  Main UI Component: Standardized data matrix template reflecting the official DOH Manual of Operations layout. Includes a Clinical Blue button with a download icon labeled "Export Official DOH Report".  🔒 Hard Data Linking Rules for PrototypingWhen you wire your prototyping click-flows together in Figma, ensure the visual data lineage remains unified across all screens.
