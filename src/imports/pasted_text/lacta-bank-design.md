Complete Figma UI/UX Design Prompt: LACTA BANK🌐 Project Context & SetupSystem Name: LACTA BANK – Makati Milk Bank Management System (MMBMS)   Platform Target: Web-Based Desktop Application designed for a 1440px viewport grid layout, fully responsive across standard desktop and mobile browsers (Chrome, Safari, Firefox).  Design Core: Digital replacement for manual milk bank logbooks, optimizing strict biological traceability (biovigilance) from a donor’s raw intake through high-precision laboratory validation down to an infant's prescription fulfillment.  🎨 Color Palette & Visual Tokens (Blue & Pink Theme)1. Branding & ArchitecturePrimary / Clinical Blue (#1E3A8A or equivalent): Establishes an environment of data integrity and corporate medical security. Use for the main sidebar background, data table header fills, modal frames, and primary system execution triggers (e.g., "Save Log to Database").  Accent / Maternal Pink (#EC4899 or equivalent): Establishes human warmth, patient interaction, and community integration. Use for donor registry buttons, active navigation markers, selection states, and dashboard metric counts.Neutral Background Tones: Plain white (#FFFFFF) and light cool gray-blue (#F8FAFC) to maximize readability across dense database tables and complex forms.2. Specialized Medical Lifecycle BadgesTo keep the theme visually clean without losing focus of critical medical warning protocols, map system workflow state indicators precisely to these variant colors:Raw Milk / Incoming: Light Pastel Pink badge with deep pink text.  Quarantined Batch: Soft Amber / Muted Yellow badge indicating hold for lab analysis or temperature drops.  Pasteurized / Passed: Soft Teal / Cyan Blue badge with dark blue text, marking inventory as completely safe for neonatal prescription fulfillment.  Rejected / Discarded: Muted Coral Red badge for expired lots, failed tests, or critical data validation anomalies.  🧭 Application Shell & Unified Navigation StructureProvide a persistent, high-density Fixed Left Navigation Sidebar that dynamically maps user capabilities using Role-Based Access Control (RBAC) tokens:  Admin/Manager View: Dashboard, User Management Console , Audit Trail, System Report Engine.  Nurse/Staff View: Donor Records, Intake Station, Lab Results Panel, Pasteurization Manager, Dispensing Portal.  Donor View: Minimalist workspace to view donation metrics and submit immediate health updates.  Beneficiary View: Customer dashboard tracking inventory availability and submitted requests.  🖥️ Modular Screen-by-Screen UI/UX BlueprintsScreen 1: Access Control Gateway (UC-01)Layout: Clean, centered credential card floating on a light blue-gray background.  Forms: Target labels for Email/Username and Encrypted Password.  Interactivity: Focused text boxes trigger a glowing Maternal Pink outline. The submission action is a solid Clinical Blue button titled "Authenticate Session".Screen 2: Donor Onboarding Portal (UC-02 & UC-03)Directory Grid: A tabular directory showing the DONORS schema (DTN, First Name, Last Name, Medical Status, Operational Program, and Eligibility State).  Onboarding Drawer: Slides open from the right side upon clicking a pink "+ Register Donor" action block. Captures informed consent toggles and sets the initial core operation stream: Supsup Todo, Milky Way, or Mom's Act.  Health Updates Module: View with inputs to track serological evaluation vectors (HIV/Hepatitis B updates). High-risk flags must visually transform the row item into a Temp_Deferred highlight.  Screen 3: Milk Intake Registry (UC-09)Form UI: Specialized text fields optimized for quick clinical entry mapping the MILK COLLECTION framework.  Interactive Elements:DTN validation lookup field.  Volume inputs (numeric field displaying an internal suffix tracking unit in $\text{mL}$).  Arrival temperature inputs (numeric field displaying an internal suffix tracking unit in $\degree\text{C}$).  Error Trigger State: Design a component variant showing a red-bordered validation warning if input volumes breach the safe ceiling threshold of 800mL per day.  Screen 4: Laboratory Analysis Panel (UC-10)Interface: Form structure displaying test parameters grouped by tracking sequence phase tabs: Pre-Pasteurization vs. Post-Pasteurization.  Data Fields: Input fields for Total Plate Counts, Coliform Pathogen Counts, and Chemical pH Balance indicators.  Validation States: Include binary check components for Pass or Fail status evaluations that determine product movement.  Screen 5: Pasteurization Master Command (UC-04)Batch Assembler: Multi-select checklist workspace to pull available entries marked as 'Raw' into a brand new combined processing lot (Batch ID).  Holder Method Thermal Log: Embedded tabular module tracking required processing criteria : Start Time, End Time, Bottle Count, and 5-minute interval tracking checks confirming a steady range of $62.5\degree\text{C} \rightarrow 63\degree\text{C}$ for 30 minutes.  Constraint UI: The transition control button to mark an inventory lot as ready must remain locked if an interval text field is incomplete or falls out of bounds.  Screen 6: Live Inventory & Critical Warning Board (UC-05)KPI Metrics Block: Three dominant top metrics styled with a soft gradient blending Clinical Blue to Maternal Pink at 12% opacity. Displays active volume totals across: Raw Milk, Quarantined Lots, and Pasteurized Ready Stock.  Inventory Tracker Grid: Real-time data lists reflecting calculated Expiration Date parameters.  Visual Banners: Prominent floating warning toasts alerting managers when a category hits a Low-Stock Alert or when batches approach a 30-Day Expiry Window.  Screen 7: Beneficiary Queue & Fulfillment Desk (UC-06 & UC-08)Profile Setup Panel: Patient directory tracking Guardian Details, Clinic Assignment, active Pediatrician Prescription checkboxes, and insulated cooler transit checkmarks.  Sorting Hierarchy: Visual pill badges assigning urgency tags: Critical/NICU, High, or Standard.  FEFO Checkout Form: An ordering window tracking the target requested volume. It features an auto-calculated Processing Fee cell item displaying values in currency units ($\text{PHP}$) and handles billing options across Paid, Pending, or Waived states.  Screen 8: Email Communications Automation Hub (UC-07)Logs View: Table displaying history metrics from the EMAILING LOGS schema (Email ID, Beneficiary ID, Target Batch ID, Status, and Sender Account reference).  Template Sandbox Overlay: Interactive mockup popup window visualizing a stylized HTML email dispatch containing the official LACTA BANK header, automatic pickup timeline tables, and real-time clearance distribution details.  Screen 9: Analytics & Reporting Matrix (UC-11)Control UI: Segmented range tabs allowing administrators to toggle interval compilation loops across Weekly, Monthly, and Annual metrics.  Layout Design: Structured document card layout mimicking the standardized layout metrics of the official Department of Health (DOH) Manual of Operations, featuring a Clinical Blue "Download Signed PDF" tool component.  Screen 10: Security Console & System Footprints (UC-12 & UC-13)RBAC Controls: Active credential modification forms allowing an Admin to change user assignments or instantly revoke a server operational profile.  Audit Trail Viewer: Read-only historical chronological log table displaying AUDIT LOGS parameter entries (Log ID, Changed By, Audit Action, Target Table, Old Data Snapshot string, and New Data Overwrite string).  🔒 Prototyping Linkage & Safety RulesWhen stringing together screen-to-screen click-actions in Figma, ensure the visual navigation path preserves your system’s multi-layered relational database dependencies:

{USERS (RBAC Access Cleared)} --> {DONORS (DTN)} --> {MILK COLLECTION (CTN)}-->{PASTEURIZATION LOGS (Batch ID)}

{PASTEURIZATION LOGS (Batch ID)} --> {LAB RESULTS (Status: Pass Gateway)} --> {DISPENSING LOGS} --> {BENEFICIARIES}

If a user clicks a path in your prototype run that violates biovigilance protocols (e.g., trying to dispense a batch whose Post-Pasteurization lab status is 
not marked as an explicit Pass), make sure an error banner overlays the page layout saying: "Operation Suspended: Laboratory clearance required to protect product lineage integrity."
